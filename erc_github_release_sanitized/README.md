# Reliability-Aware Visual ERC (Sanitized Release)

This repository contains a GitHub-safe research code template for a two-stage multimodal Emotion Recognition in Conversation (ERC) framework over MELD and IEMOCAP.

No private absolute paths, personal usernames, local machine names, API keys, checkpoints, raw media, extracted features, logs, or prediction files are included.

## Method summary

### Stage 1: Semantically guided visual affective cue localization

A frozen VLM extracts a 4096-D visual affective feature from sampled video frames. Optional prompts include:

- target-speaker localization guidance,
- dialogue context,
- acoustic descriptions generated by an audio-language model,
- lexical VAD priors.

The VLM extractor template is in:

```bash
feature_tools/extract_vlm_embeddings_template.py
```

### Stage 2: Reliability-aware affective complementation

The classifier uses pre-extracted text, audio, and visual features:

- text: 1024-D utterance representation extracted with RoBERTa / EmoBERTa,
- audio: 1024-D frame-level sequence, usually padded/truncated to 300 frames,
- video: 4096-D global VLM embedding.

The model projects all modalities to a 512-D hidden space, applies past-only context modeling, uses video as the query to retrieve text/audio affective references, and updates the visual representation with a confidence-gated residual.

## Text feature extraction

The repository includes an EmoBERTa-style text feature extractor inspired by the public `tae898/erc` implementation. The script builds each text input from the current utterance plus optional past/future utterances and optional speaker-name prefixes, then exports one CLS embedding per utterance.

MELD example:

```bash
python feature_tools/extract_text_emoberta_features.py \
  --dataset meld \
  --meld_root /path/to/MELD.Raw \
  --model_name_or_path roberta-large \
  --output_dir /path/to/meld/features/wenben_sorted \
  --split all \
  --speaker_mode none \
  --num_past_utterances 0 \
  --num_future_utterances 0 \
  --gpu 0
```

IEMOCAP example:

```bash
python feature_tools/extract_text_emoberta_features.py \
  --dataset iemocap \
  --iemocap_csv_root /path/to/iemocap/csv \
  --model_name_or_path roberta-large \
  --output_dir /path/to/iemocap/features/wenben \
  --split all \
  --speaker_mode none \
  --num_past_utterances 0 \
  --num_future_utterances 0 \
  --gpu 0
```

The output format is compatible with `erc/datasets.py`: `features`, `labels`, and `ids`.

## Configuration

Copy an example config and edit only local paths:

```bash
cp configs/meld.example.json configs/meld.local.json
cp configs/iemocap.example.json configs/iemocap.local.json
```

The `*.local.json` files should not be committed if they contain local private paths.

## Training

MELD:

```bash
python train.py \
  --config configs/meld.local.json \
  --dataset meld \
  --modalities text,audio,video \
  --gpu 0 \
  --epochs 10 \
  --lr 8e-6 \
  --dropout 0.4 \
  --aux_loss_weight 0.15 \
  --context_mode past
```

IEMOCAP:

```bash
python train.py \
  --config configs/iemocap.local.json \
  --dataset iemocap \
  --modalities text,audio,video \
  --gpu 0 \
  --epochs 30 \
  --lr 4e-5 \
  --dropout 0.45 \
  --aux_loss_weight 0.20 \
  --context_mode past
```

## Ablations

```bash
# no contrastive loss
python train.py --config configs/meld.local.json --dataset meld --disable_cl

# no reliability gate
python train.py --config configs/meld.local.json --dataset meld --no_conf_gate

# no context modeling
python train.py --config configs/meld.local.json --dataset meld --disable_context

# no cross-modal residual complementation
python train.py --config configs/meld.local.json --dataset meld --disable_crossmodal

# remove audio-to-video interaction only
python train.py --config configs/meld.local.json --dataset meld --disable_audio_video_interaction

# remove text-to-video interaction only
python train.py --config configs/meld.local.json --dataset meld --disable_text_video_interaction
```

## Privacy and release notes

This release replaces all original hard-coded local paths with config or CLI arguments. It also excludes private datasets, raw video/audio, extracted features, logs, checkpoints, generated predictions, and model weights.

Before uploading to GitHub, run a local privacy scan for absolute paths, usernames, emails, credentials, checkpoints, raw media, and generated prediction files.
